PartyRock — App Prototyping

PartyRock is AWS's no-code AI playground. You describe an app, drag widgets onto a canvas, write prompts, and in minutes you have a working AI prototype — text generators, image generators, chatbots, file analyzers. All powered by Amazon Bedrock behind the scenes.

What it's great for: Testing ideas fast. Figuring out exactly what prompts and widgets your app needs — without writing a single line of code.

What it can't do: Let anyone use your app without a PartyRock account. Give you a custom URL. Let you control the design. Scale beyond the playground.

Kiro — Agentic AI Development Tool

Kiro is an AI-powered IDE that acts as your development partner. You describe what you want to build — in plain English — and Kiro reads your PartyRock prototype, understands every widget and prompt, and writes all the real production code: Lambda functions, API Gateway config, SAM infrastructure templates, GitHub Actions deployment workflows, and a complete frontend.

What makes it agentic: Kiro doesn't just autocomplete. It plans, reasons, and executes multi-step tasks. You give it your PartyRock widget map and screenshots, and it produces a complete, deployable application. You never write a line of code.

Amazon Bedrock + AWS Services — Published Working App

Amazon Bedrock is the AI engine that PartyRock uses under the hood. When you "slice" your PartyRock app, your Lambda functions call Bedrock directly — the same Claude models, the same quality, but now running in infrastructure you own and control.

What Is Amazon Bedrock? (Read this first)

Foundation Models

Amazon Bedrock is a fully managed AWS service that gives you access to foundation models — large AI models trained on massive amounts of data that can understand and generate text, images, and more.

Think of a foundation model like a very capable brain you can rent. You send it a question or instruction. It sends back a response. You pay for what you use. You don't manage any servers.

ProviderModels available via Bedrock
AnthropicClaude Sonnet 4.5 (used in this guide), Claude 3 Haiku, Claude 3 Sonnet
AmazonTitan Text, Nova Canvas (images)
MetaLlama 3
Stability AIStable Diffusion XL (images)

The Bedrock API — What It Actually Looks Like

Here is the core of what Bedrock does. Your code sends a request like this:

POST /model/anthropic.claude-3-sonnet.../invoke

{
  "messages": [
    { "role": "user", "content": "What are the calories in a chicken salad?" }
  ],
  "max_tokens": 1024,
  "temperature": 0.7
}

Bedrock routes it to Claude and returns:

{
  "content": [
    { "text": "A typical chicken salad contains approximately 350–450 calories..." }
  ]
}

That's it. That's Amazon Bedrock. An API that connects your code to powerful AI models.

Key Concepts Glossary

TermPlain English
Foundation ModelA large pre-trained AI model (Claude, Llama, Titan). You don't train it — you call it via API.
InferenceThe act of running a model to get a response. Every time a user clicks Run All, Bedrock performs inference.
TokenThe unit Bedrock charges by. Roughly 4 characters = 1 token. A typical AI response = 200–500 tokens ≈ $0.001–0.003.
TemperatureControls creativity vs. consistency. 0 = always the same answer. 1 = more varied. Maps to the Temperature slider in PartyRock's Advanced tab.
Top PControls word choice diversity. Lower = more focused. Higher = more variety. Maps to the Top P slider in PartyRock's Advanced tab.
Model IDThe unique identifier for a specific model version, e.g. us.anthropic.claude-sonnet-4-5-20250929-v1:0. The us. prefix indicates a cross-region inference profile.

Look Inside the Lambda — The Actual Bedrock API Call

Before moving on, open one of the Lambda .py files Kiro generated (in the file explorer on the left, under backend/lambda/). Find the bedrock.invoke_model_with_response_stream() call. It looks like this:

response = bedrock.invoke_model_with_response_stream(
    modelId="us.anthropic.claude-sonnet-4-5-20250929-v1:0",
    body=json.dumps({
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 1024,
        "messages": [{"role": "user", "content": prompt}]
    })
)

Why streaming? The standard invoke_model() waits for the full response before returning. invoke_model_with_response_stream() returns tokens as they are generated — so users see the AI "thinking" in real time instead of waiting for a blank screen.

Connect this back to what you saw in PartyRock:

Code lineWhere it came from
modelId="us.anthropic.claude-sonnet-4-5..."The model you selected in PartyRock's AI model tab. The us. prefix is required for the cross-region inference profile.
"content": promptYour PartyRock prompt, with widget values injected at runtime
"max_tokens": 1024Limits response length — roughly 750 words maximum
"temperature" (if present)The Temperature slider value from PartyRock's Advanced tab
This is Amazon Bedrock. One streaming API call. Your prompt goes in. Claude's response comes out token by token. Everything else in this guide is infrastructure to make that call accessible to anyone on the internet.

Step 4 — Enable Bedrock AI Models (5 min) Bedrock

Amazon Bedrock is the AI engine that PartyRock uses under the hood. Now you enable it in your own AWS account so your Lambda functions can call it directly.

  1. Search for Bedrock → confirm region is US East (N. Virginia) us-east-1
  2. Left sidebar → Model access → Manage model access
  3. Under Anthropic, check Claude Sonnet 4 (Claude Sonnet 4.5) → Save changes
  4. Refresh until status shows Access granted ✅
This is the same Claude model your PartyRock app uses. The model ID is us.anthropic.claude-sonnet-4-5-20250929-v1:0 — a cross-region inference profile. The us. prefix is required and is already included in the code Kiro generates.
Why do you have to request access? Bedrock requires explicit opt-in for each foundation model. This ensures you've acknowledged the model's usage policies and gives AWS visibility into which models are being used. Once granted, any Lambda function with AmazonBedrockFullAccess can call this model.
Model access can take up to 5–10 minutes to activate. If the status still shows "Requested" after a few minutes, refresh the page and wait. Do not proceed to Step 5 until it shows "Access granted".
  • In us-east-1 region
  • Claude Sonnet 4 (Claude Sonnet 4.5) shows "Access granted"

What Happens When You Click Run All

Here is the complete request flow — every step from your click to the AI response:

3LambdaRuns your Python code — builds the prompt by injecting input values, calls bedrock.invoke_model_with_response_stream()
4Amazon BedrockReceives the API call, routes it to Claude Sonnet 4.5
5Claude (inference)Processes your prompt and generates a response — this is the AI step
6Bedrock → LambdaReturns the response JSON to Lambda
7Lambda → API GatewayExtracts the text, returns it with CORS headers
8BrowserDisplays the response in the output panel
